# 项目完善说明：改前 / 改后

## 1. 后端架构

### 改前
项目只有 React 前端，业务数据主要写在 `mockVideos`、`mockUsers`、`mockRooms` 里面，没有真正的 Python 后端，也没有真正连接 PostgreSQL。

### 改后
新增：

```text
backend/app/main.py
backend/app/models.py
backend/app/schemas.py
backend/app/security.py
backend/app/database.py
backend/schema.sql
backend/requirements.txt
```

实现了 FastAPI + PostgreSQL 后端，支持登录注册、视频查询、搜索、上传、审核、评论、弹幕、直播间创建和 WebSocket 直播聊天。

---

## 2. 登录注册

### 改前：`src/stores/authStore.ts`

```ts
const mockUsers = { admin: ..., creator: ..., user: ... };
login: async (account, password) => {
  const mockUser = mockUsers[account];
  if (mockUser && mockUser.password === password) {
    set({ token: 'mock_token_' + account, user: mockUser.user });
  }
}
```

### 改后：`src/stores/authStore.ts`

```ts
const data = await apiRequest('/api/auth/login', {
  method: 'POST',
  body: JSON.stringify({ account, password })
});
set({ token: data.token, user: data.user, isLoggedIn: true });
```

登录、注册和修改资料都走 Python 后端。

---

## 3. 视频数据

### 改前：`src/stores/videoStore.ts`

```ts
const mockVideos: Video[] = [...];
fetchVideos: async () => {
  set({ videos: mockVideos });
}
```

### 改后：`src/stores/videoStore.ts`

```ts
const data = await apiRequest(`/api/videos?${query.toString()}`);
set({ videos: data.items, hasMore: data.hasMore });
```

首页、发现页、搜索页都从 PostgreSQL 读取已审核视频。

---

## 4. 上传视频

### 改前：`src/pages/UploadPage.tsx` + `videoStore.ts`

```ts
uploadVideo: async () => {
  await new Promise(resolve => setTimeout(resolve, 1000));
  return true;
}
```

上传成功后并不会真的保存，也不会出现在首页或后台。

### 改后

```ts
const created = await apiRequest('/api/videos', {
  method: 'POST',
  body: JSON.stringify(videoData)
});
```

上传后视频进入 PostgreSQL，状态为 `audit_status = 0`，需要管理员审核。

---

## 5. 管理员审核

### 改前：`src/pages/AdminPage.tsx`

```ts
const mockVideos = [...];
<button>通过</button>
<button>驳回</button>
```

按钮只是显示，没有真实作用。

### 改后

```ts
const pendingVideos = await fetchPendingVideos();
await auditVideo(video.id, 1); // 通过
await auditVideo(video.id, 2); // 驳回
```

管理员通过后，视频会出现在首页推荐流。

---

## 6. 评论和弹幕

### 改前
评论和弹幕只存在前端状态里，刷新页面就丢失。

### 改后

```ts
POST /api/videos/{video_id}/comments
GET  /api/videos/{video_id}/comments
POST /api/videos/{video_id}/danmaku
GET  /api/videos/{video_id}/danmaku
```

评论和弹幕保存到 PostgreSQL。

---

## 7. 页面权限

### 改前
`/admin`、`/creator`、`/upload` 等页面可以直接访问。

### 改后
新增：

```text
src/components/auth/RequireAuth.tsx
```

并在 `src/App.tsx` 中使用：

```tsx
<Route path="/admin" element={<RequireAuth minUserType={2}><AdminPage /></RequireAuth>} />
<Route path="/creator" element={<RequireAuth minUserType={1}><CreatorPage /></RequireAuth>} />
<Route path="/upload" element={<RequireAuth minUserType={1}><UploadPage /></RequireAuth>} />
```

普通用户不能进后台，未登录用户不能上传。

---

## 8. 视频卡片跳转

### 改前

```tsx
<VideoCard video={video} />
```

点击视频卡片没有进入播放页。

### 改后

```tsx
<VideoCard video={video} onClick={() => navigate(`/video/${video.id}`)} />
```

首页、发现页、搜索页都支持点击进入播放页。

---

## 9. 直播功能

### 改前
直播间数据和聊天主要是 mock，WebSocket 地址写死。

### 改后

```ts
POST /api/live/rooms
GET  /api/live/rooms
GET  /api/live/rooms/{room_id}
WS   /ws/live/{room_id}
```

创作者可以创建直播间，用户进入直播间后可通过 WebSocket 发送实时聊天弹幕。
