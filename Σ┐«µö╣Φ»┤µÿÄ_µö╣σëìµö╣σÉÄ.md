# 项目完善说明：改前 / 改后

## 1. 后端架构

### 改前
项目只有 React 前端，业务数据主要写在 `mockVideos`、`mockUsers`、`mockRooms` 里面，没有真正的 Python 后端，也没有真正连接 PostgreSQL。

### 改后
新增：

```text
backend/app/main.py
backend/app/models.py
backend/app/schemas.py
backend/app/security.py
backend/app/database.py
backend/schema.sql
backend/requirements.txt
```

实现了 FastAPI + PostgreSQL 后端，支持登录注册、视频查询、搜索、上传、审核、评论、弹幕、直播间创建和 WebSocket 直播聊天。

## 2. 登录注册

### 改前：`src/stores/authStore.ts`

```ts
const mockUsers = { admin: ..., creator: ..., user: ... };
login: async (account, password) => {
  const mockUser = mockUsers[account];
  if (mockUser && mockUser.password === password) {
    set({ token: 'mock_token_' + account, user: mockUser.user });
  }
}
```

### 改后：`src/stores/authStore.ts`

```ts
const data = await apiRequest('/api/auth/login', {
  method: 'POST',
  body: JSON.stringify({ account, password })
});
set({ token: data.token, user: data.user, isLoggedIn: true });
```

登录、注册和修改资料都走 Python 后端。

## 12. 社区互动:用户私聊(WebSocket 实时推送)

### 改前
没有任何私聊功能，用户之间无法直接交流。

### 改后

新增数据库表 `conversations` 和 `messages`：

```sql
CREATE TABLE conversations (
    id UUID PRIMARY KEY,
    user_a_id UUID NOT NULL,           -- 强制 user_a_id < user_b_id
    user_b_id UUID NOT NULL,
    last_message_id UUID,
    last_message_at TIMESTAMPTZ,
    UNIQUE(user_a_id, user_b_id)
);

CREATE TABLE messages (
    id UUID PRIMARY KEY,
    conversation_id UUID NOT NULL,
    sender_id UUID NOT NULL,
    receiver_id UUID NOT NULL,
    content TEXT NOT NULL,
    message_type SMALLINT DEFAULT 0,   -- 0文本 1图片 2表情
    is_recalled BOOLEAN,
    is_read BOOLEAN,
    created_at TIMESTAMPTZ,
    recalled_at TIMESTAMPTZ
);
```

新增后端接口和 WebSocket 通道：

```text
GET    /api/chat/conversations                  我的会话列表(含未读数)
POST   /api/chat/conversations                  创建/获取与某人的会话
GET    /api/chat/conversations/{id}/messages   拉取历史消息(分页)
POST   /api/chat/conversations/{id}/messages   HTTP 兜底发消息
POST   /api/chat/messages/{id}/recall          撤回(2分钟内)
POST   /api/chat/conversations/{id}/read       标记会话已读
WS     /ws/chat?token=xxx                       WebSocket 主通道
```

参考已有 `LiveHub` 实现了 `ChatHub`，按用户 ID 维护连接（支持同一用户多端同时在线）：

```python
class ChatHub:
    user_conns: Dict[user_id, List[WebSocket]]

    async def push_to_user(user_id, payload): ...
```

WebSocket 消息协议：

```json
// 发送
{"type": "send", "peerId": "xxx", "content": "你好", "messageType": 0}

// 接收
{"type": "message", "data": {...}}        // 收到新消息
{"type": "recall", "messageId": "xxx"}    // 对方撤回
{"type": "read", "conversationId": "xxx"} // 对方已读
{"type": "typing", "fromUserId": "xxx"}   // 对方正在输入
```

前端：

- 新增 `src/api/chat.ts`、`src/stores/chatStore.ts`：WS 连接、消息/会话/已读全部状态管理
- 新增 `src/pages/MessagePage.tsx`：左侧会话列表(带未读角标) + 右侧聊天窗(气泡、撤回、已读状态、滚动加载)
- 新增 `src/components/notification/MessageButton.tsx`：导航栏私信入口 + 红点
- 用户主页加 "发消息" 按钮：点击跳 `/messages?peer={userId}` 自动打开与对方的会话
- 注册路由 `/messages`

### 改前

```sql

CREATE TABLE comment_mentions (
    id UUID PRIMARY KEY,
    comment_id UUID NOT NULL,
    mentioned_user_id UUID NOT NULL,
    UNIQUE(comment_id, mentioned_user_id)
);
```

后端 `POST /api/videos/{id}/comments` 增强：

```python
parent_id, reply_to_user_id = ...  # 支持二级回复
mentioned = parse_mentions(content)  # 用正则 @[\w一-龥]+ 解析

# 自动发送三类通知:
```

新增接口 `GET /api/comments/{id}/replies` 按需拉取回复列表。

## 15. Docker 开发体验:挂载 + 热重载

### 改前
后端、前端代码都是 build 时 COPY 进镜像，每次改完代码需要 `docker compose up -d --build backend` 重新构建,等 1-2 分钟。

### 改后

`docker-compose.yml` 增加挂载和热重载:

```yaml
backend:
  environment:
    WATCHFILES_FORCE_POLLING: "true"   # Windows Docker 文件监听必备
  volumes:
    - ./public:/app/public
    - ./backend/app:/app/backend/app   # 源码挂载
  command: python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload --reload-dir /app/backend/app

frontend:
  volumes:
    - ./src:/app/src
    - ./public:/app/public
    - ./webpack.config.js:/app/webpack.config.js
    - ...
```

并修改 `webpack.config.js` 打开 `liveReload: true`、`usePolling: true`,让 Windows 下文件改动能被检测到。

修改任何后端 Python 或前端 React 源码,**无需重启容器**,浏览器自动刷新。
