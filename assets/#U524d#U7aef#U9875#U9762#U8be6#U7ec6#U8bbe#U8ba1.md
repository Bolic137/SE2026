# 前端页面详细设计

## 1 页面路由规划

| 路径 | 页面 | 权限 | 说明 |
|:---|:---|:---:|:---|
| `/` | 首页 | 公开 | 推荐视频流 |
| `/discover` | 发现页 | 公开 | 分类浏览 |
| `/search` | 搜索结果页 | 公开 | 关键词搜索 |
| `/video/:id` | 视频播放页 | 公开 | 播放器+弹幕+评论 |
| `/live/:roomId` | 直播间页 | 公开 | 直播播放+弹幕互动 |
| `/live/start` | 开播页 | 创作者 | 配置直播间+获取推流地址 |
| `/upload` | 视频上传页 | 创作者 | 分片上传+信息填写 |
| `/user/:id` | 用户主页 | 公开 | 用户信息+视频列表 |
| `/creator` | 创作者中心 | 创作者 | 内容管理+数据分析 |
| `/settings` | 个人设置 | 登录 | 修改个人信息 |
| `/login` | 登录页 | 公开 | 弹窗或独立页 |
| `/register` | 注册页 | 公开 | 弹窗或独立页 |
| `/admin` | 管理后台 | 管理员 | 内容审核+用户管理 |

## 2 全局布局组件

### 2.1 AppLayout（根布局）

```
┌─────────────────────────────────────────┐
│  TopNav（顶部导航栏，固定高度60px）       │
├─────────────────────────────────────────┤
│                                         │
│         RouterView（页面内容区）         │
│                                         │
└─────────────────────────────────────────┘
```

- TopNav：全站固定，含 Logo、搜索框、用户入口
- 无侧边栏：参考抖音/YouTube，内容区全宽，减少组件复杂度

### 2.2 TopNav（顶部导航栏）

| 区域 | 内容 | 说明 |
|:---|:---|:---|
| 左侧 | Logo（点击回首页） | 文字Logo即可，无需图片 |
| 中间 | 搜索输入框 | 回车触发搜索，跳转 `/search` |
| 右侧 | 未登录：登录/注册按钮；已登录：头像下拉菜单 | 下拉菜单含：个人主页、创作者中心、设置、退出 |

### 2.3 弹窗组件

| 弹窗 | 触发方式 | 内容 |
|:---|:---|:---|
| LoginModal | 点击"登录" | 账号输入、密码输入、登录按钮、去注册链接 |
| RegisterModal | 点击"注册" | 账号、密码、验证码输入、注册按钮 |
| UploadModal | 点击"上传"（创作者） | 直接跳转 `/upload`，不用弹窗 |

## 3 各页面详细设计

### 3.1 首页 `/`

**布局**：
```
┌─────────────────────────────────────────┐
│              TopNav                      │
├─────────────────────────────────────────┤
│  CategoryBar（分类标签栏，横向滚动）      │
├─────────────────────────────────────────┤
│                                         │

│     （滚动到底部自动加载下一页）            │
│                                         │
└─────────────────────────────────────────┘
```

**组件拆分**：

| 组件 | 说明 |
|:---|:---|
| CategoryBar | 横向滚动标签栏，展示分类，点击筛选，"推荐"默认选中 |
| VideoCard | 封面图(16:9) + 标题(2行截断) + 作者头像+昵称 + 播放量 |

- 分类切换 → 调 `/api/videos?categoryId=x&page=1` → 重置列表
- 触底加载 → page++，追加数据

### 3.2 视频播放页 `/video/:id`

**布局**：
```
┌─────────────────────────────────────────────────────────┐
│                      TopNav                              │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌─────────────────────────────┐  ┌─────────────────┐ │
│  │                             │  │  UpInfo（UP主）  │ │
│  │   VideoPlayer（播放器）      │  │  头像+昵称+关注   │ │
│  │   + DanmakuLayer（弹幕层）   │  ├─────────────────┤ │
│  │                             │  │  RelatedVideos   │ │
│  │   Controls（控制栏）         │  │  相关视频推荐     │ │
│  │                             │  │  （竖排5个卡片）  │ │
│  └─────────────────────────────┘  └─────────────────┘ │
│                                                         │
│  VideoInfo（标题+标签+播放量+时间）                     │
│  ActionBar（点赞/收藏/分享按钮）                         │
│  ─────────────────────────────────                     │
│  CommentSection（评论区）                               │
│  ┌─────────────────────────────────┐                   │
│  │  CommentInput（输入框+发送）     │                   │
│  │  CommentList（评论列表）         │                   │
│  │  ├─ CommentItem（顶层评论）      │                   │
│  │  │  └─ ReplyList（回复列表）     │                   │
│  │  └─ ...                         │                   │
│  └─────────────────────────────────┘                   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**组件拆分**：

| 组件 | 说明 |
|:---|:---|
| VideoPlayer | DPlayer 实例，16:9 比例，宽度自适应，高度自动计算 |
| DanmakuLayer | DPlayer 自带弹幕层，配置弹幕 API 地址 |
| VideoInfo | 标题、标签（可点击跳转搜索）、播放量、发布时间 |
| ActionBar | 点赞（带数量）、收藏、分享按钮，点击调对应 API |
| UpInfo | UP 主头像+昵称+粉丝数，关注/取消关注按钮 |
| RelatedVideos | 右侧相关视频，调 `/api/videos/related?videoId=x` |
| CommentInput | 文本域 + 发送按钮，未登录提示登录 |
| CommentItem | 用户头像+昵称+内容+时间+点赞数+回复按钮 |
| ReplyList | 缩进展示回复，最多展开 3 条，更多点击展开 |

**数据流**：
- 页面加载 → 并行调：视频详情、播放地址、弹幕列表、评论列表、相关视频
- 播放进度 → 定时（5秒）上报 `/api/videos/history`（记录观看进度）
- 发送弹幕 → WebSocket 发送，同时调 `/api/danmaku` 持久化

```
┌─────────────────────────────────────────────────────────┐
│                      TopNav                              │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌─────────────────────────────┐  ┌─────────────────┐ │
│  │                             │  │  RoomInfo        │ │
│  │   LivePlayer（直播播放器）   │  │  标题+主播+在线    │ │
│  │   flv.js 渲染               │  │  人数+分类       │ │
│  │                             │  ├─────────────────┤ │
│  └─────────────────────────────┘  │  ChatPanel       │ │
│                                   │  （聊天+弹幕区）  │ │
│                                   │  ┌─────────────┐ │ │
│                                   │  │ ChatMessage  │ │ │
│                                   │  │ 列表（滚动）  │ │ │
│                                   │  └─────────────┘ │ │
│                                   │  ┌─────────────┐ │ │
│                                   │  │ ChatInput    │ │ │
│                                   │  │ 输入+发送    │ │ │
│                                   │  └─────────────┘ │ │
│                                   └─────────────────┘ │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**组件拆分**：

| 组件 | 说明 |
|:---|:---|
| LivePlayer | flv.js 播放器，FLV 格式拉流，16:9 比例 |
| RoomInfo | 直播间标题、主播信息、在线人数、直播状态标签 |
| ChatPanel | 右侧聊天区域 |
| ChatMessage | 单条消息：用户昵称+内容，系统消息居中灰色 |
| ChatInput | 输入框+颜色选择器+发送按钮，WebSocket 发送 |

**数据流**：
- 页面加载 → 调 `/api/live/rooms/:id` 获取信息 → flv.js 开始拉流
- WebSocket 连接 → `ws://host/ws/live/:roomId`
- 进入房间 → 发送 `{"type":"join","roomId":x}`
- 发送弹幕 → 发送 `{"type":"danmaku","content":"...","color":"#FFF"}`
- 接收消息 → 服务器广播，ChatMessage 列表追加
- 离开页面 → WebSocket 关闭

### 3.5 视频上传页 `/upload`

**布局**：
```
┌─────────────────────────────────────────┐
│              TopNav                      │
├─────────────────────────────────────────┤
│                                         │
│  Step1: UploadArea（文件上传区）         │
│  ┌─────────────────────────────────┐    │
│  │  拖拽视频至此或点击上传          │    │
│  │  （支持MP4/AVI/MOV，最大2GB）    │    │
│  └─────────────────────────────────┘    │
│         ↓                               │
│  Step2: UploadProgress（上传进度）       │
│  ┌─────────────────────────────────┐    │
│  │  进度条 + 百分比 + 已传/总大小   │    │
│  └─────────────────────────────────┘    │
│         ↓                               │
│  Step3: VideoInfoForm（信息填写）        │
│  ┌─────────────────────────────────┐    │
│  │ 标题输入框                       │    │
│  │ 描述文本域                       │    │
│  │ 标签输入（支持回车添加，自动补全）│    │
│  │ 分类下拉选择                     │    │
│  │ 封面选择（自动截取第5秒/本地上传）│    │
│  │ 发布按钮                         │    │
│  └─────────────────────────────────┘    │
│                                         │
└─────────────────────────────────────────┘
```

**组件拆分**：

| 组件 | 说明 |
|:---|:---|
| UploadArea | Element Plus Upload 组件，拖拽+点击，accept="video/*" |
| UploadProgress | 分片上传进度，每片 5MB，失败重传单分片 |
| VideoInfoForm | 表单校验：标题必填(2-100字)、分类必选 |
| TagInput | 自定义组件，回车添加标签，最多 10 个，每个最多 10 字 |

**数据流**：
- 选择文件 → 前端切片（每片 5MB）→ 循环 POST `/api/videos/upload`（multipart，含 chunkIndex/totalChunks）
- 全部分片成功 → POST `/api/videos`（JSON，含元信息+文件名标识）→ 后端合并转码
- 发布成功 → 跳转用户主页

## 4 Pinia Store 设计

### 4.1 userStore（用户状态）

```javascript
state: {
  token: localStorage.getItem('token') || '',
  userInfo: null,  // { id, nickname, avatar, userType }
  isLoggedIn: false
}
actions: {
  login(account, password) // 调API，存token，获取用户信息
  logout() // 清token，清userInfo
  fetchUserInfo() // 调 /api/users/me
}
```

### 4.2 playerStore（播放状态）

```javascript
state: {
  currentVideo: null,
  isPlaying: false,
  currentTime: 0,
  danmakuVisible: true
}
```

### 4.3 liveStore（直播状态）

```javascript
state: {
  ws: null,           // WebSocket实例
  messages: [],       // 聊天消息列表
  onlineCount: 0,
  roomInfo: null
}
actions: {
  connect(roomId)     // 建立WebSocket连接
  sendDanmaku(data) // 发送弹幕
  disconnect()      // 关闭连接
}
```

## 7 路由守卫

| 守卫 | 逻辑 |
|:---|:---|
| 全局前置 | 检查目标路由 meta.requiresAuth，未登录弹登录窗或跳转 |
| 创作者权限 | 检查 userInfo.userType === 1，否则提示申请创作者 |
| 管理员权限 | 检查 userInfo.role === 'admin'（或单独admin token） |
