# 推荐算法设计

## 1 设计原则

- 基于内容推荐：实现简单，无需用户行为矩阵，冷启动友好
- 热度兜底：新用户/无历史时展示热门内容
- 实时性要求低：推荐列表可缓存5分钟，降低数据库压力
- 5人团队可维护：公式透明，参数可调，无需机器学习框架

## 2 推荐场景

| 场景 | 推荐源 | 说明 |
|:---|:---|:---|
| 首页推荐 | 个性化 + 热门混合 | 登录用户个性化，游客纯热门 |
| 视频播放页相关推荐 | 基于当前视频标签 | 同标签视频，排除已看 |
| 搜索结果页 | 基于关键词匹配 | 全文检索+热度排序 |
| 发现页 | 分类 + 时间倒序 | 按分类浏览最新内容 |

## 3 用户兴趣模型

### 3.1 兴趣标签存储

表：`t_user_interest`（user_id, tag, score）

- 用户观看视频 → 提取视频标签 → 对应标签 score +1
- 用户点赞视频 → 对应标签 score +3
- 用户收藏视频 → 对应标签 score +5
- score 上限 100，防止单一标签垄断

### 3.2 兴趣更新时机

```
用户观看视频超过30秒 → 异步调 InterestService.addScore(userId, videoTags, 1)
用户点赞视频 → 异步调 InterestService.addScore(userId, videoTags, 3)
用户收藏视频 → 异步调 InterestService.addScore(userId, videoTags, 5)
```

异步方式：Spring 的 `@Async` + 线程池，不阻塞主流程

## 4 视频热度计算

### 4.1 热度公式

```
score = (view_count * 1 + like_count * 3 + comment_count * 2 + favorite_count * 4) 
        / pow((hours_since_publish + 2), 1.5)
```

- 分子：加权互动分（点赞权重最高，收藏次之）
- 分母：时间衰减（小时数+2的1.5次方），新视频有优势
- 结果：浮点数，越大越热

### 4.2 热度更新策略

| 方式 | 实现 | 频率 |
|:---|:---|:---|
| 实时增量 | 点赞/评论时 Redis ZINCRBY | 实时 |
| 定时全量 | 定时任务扫描全表重新计算 | 每10分钟 |
| 新视频初始分 | 发布时赋予基础分 10 | 发布时 |

Redis 存储：`ZADD hot_videos:all score videoId`
分类热度：`ZADD hot_videos:category:{categoryId} score videoId`

## 5 推荐算法实现

### 5.1 登录用户个性化推荐

```
输入：userId, page, size
输出：视频ID列表

步骤：
1. 查 t_user_interest 获取用户 Top10 兴趣标签
2. 若标签为空 → 走热门推荐流程
3. 对每个兴趣标签：
   - 从 t_video 查该标签的视频（audit_status=1, status=0）
   - 按 upload_time 倒序，取最近100条
4. 合并所有标签结果，去重
5. 对每条视频计算匹配分：
   match_score = Σ(标签在兴趣中的score)
6. 按 match_score 降序排序
7. 分页返回
```

SQL 示例（单标签查询）：
```sql
SELECT * FROM t_video 
WHERE audit_status = 1 AND status = 0 
  AND tags LIKE '%游戏%' 
ORDER BY upload_time DESC 
LIMIT 100
```

### 5.2 热门推荐（游客/冷启动）

```
输入：categoryId(可选), page, size
输出：视频ID列表

步骤：
1. 若指定分类 → 从 Redis ZREVRANGE hot_videos:category:{id} 取Top100
2. 未指定分类 → 从 Redis ZREVRANGE hot_videos:all 取Top100
3. 过滤用户已观看（登录用户查 t_video_history）
4. 分页返回
```

### 5.3 相关视频推荐

```
输入：currentVideoId, size
输出：视频ID列表

步骤：
1. 查当前视频的标签列表
2. 对每个标签查同类视频（排除当前视频和已看）
3. 计算匹配度：共同标签数 / 总标签数
4. 按匹配度降序，取前size条
```

### 5.4 新内容发现（公平性）

```
每个分类下，最近24小时发布的视频：
- 赋予基础曝光量（首页推荐池混入20%新视频）
- 新视频热度分 = 互动分 / pow(2, 1.5) = 互动分 / 2.83

实现：
- 定时任务每10分钟生成 "fresh_videos" 列表
- 推荐接口混合：80% 个性化/热门 + 20% 新视频
```

### 6.2 缓存策略

| 缓存 | Key | TTL | 说明 |
|:---|:---|:---:|:---|
| 热门榜单 | `hot_videos:all` | 10分钟 | ZSet结构，score为热度分 |
| 分类热门 | `hot_videos:cat:{id}` | 10分钟 | 同上 |
| 用户推荐 | `rec:user:{userId}` | 5分钟 | List结构，预计算推荐列表 |
| 新视频池 | `fresh_videos` | 10分钟 | 最近24小时新视频ID列表 |

缓存失效：用户产生新行为（点赞/收藏）时，删除 `rec:user:{userId}` 强制重新计算

## 7 搜索功能

### 7.1 实现方式

MySQL 全文索引（最简单，无需引入ES）

```sql
-- 创建全文索引（已在t_video建表时定义）
FULLTEXT KEY ft_title_tags (title, tags)

-- 搜索SQL
SELECT * FROM t_video 
WHERE MATCH(title, tags) AGAINST('关键词' IN NATURAL LANGUAGE MODE)
  AND audit_status = 1 AND status = 0
ORDER BY upload_time DESC
LIMIT 0, 20
```

### 7.2 排序方式

| 排序 | SQL ORDER BY |
|:---|:---|
| 综合 | `MATCH(...) DESC, view_count DESC` |
| 最新 | `upload_time DESC` |
| 最热 | `view_count DESC` |

### 7.3 搜索接口

```
GET /api/search/videos?keyword=xxx&sort=comprehensive&page=1&size=20
```

## 9 关键代码示例

### 9.1 热度计算服务

```java
@Service
public class HotRankService {

    @Autowired
    private StringRedisTemplate redisTemplate;
    @Autowired
    private VideoMapper videoMapper;

    // 定时任务：每10分钟执行
    @Scheduled(fixedRate = 600000)
    public void refreshHotRank() {
        List<Video> videos = videoMapper.selectList(
            new QueryWrapper<Video>()
                .eq("audit_status", 1)
                .eq("status", 0)
        );

        for (Video v : videos) {
            double hours = Duration.between(v.getUploadTime(), LocalDateTime.now()).toHours() + 2;
            double score = (v.getViewCount() * 1 + v.getLikeCount() * 3 
                          + v.getCommentCount() * 2 + v.getFavoriteCount() * 4) 
                          / Math.pow(hours, 1.5);

            redisTemplate.opsForZSet().add("hot_videos:all", v.getId().toString(), score);
            redisTemplate.opsForZSet().add("hot_videos:cat:" + v.getCategoryId(), v.getId().toString(), score);
        }

        // 只保留Top500，节省内存
        redisTemplate.opsForZSet().removeRange("hot_videos:all", 0, -501);
    }
}
```

### 9.2 推荐服务

```java
@Service
public class RecommendService {

    @Autowired
    private UserInterestMapper interestMapper;
    @Autowired
    private VideoMapper videoMapper;
    @Autowired
    private StringRedisTemplate redisTemplate;

    public List<VideoVO> recommend(Long userId, int page, int size) {
        // 1. 查缓存
        String cacheKey = "rec:user:" + userId;
        List<String> cached = redisTemplate.opsForList().range(cacheKey, (page-1)*size, page*size-1);
        if (cached != null && !cached.isEmpty()) {
            return fetchVideosByIds(cached);
        }

        // 2. 查用户兴趣
        List<UserInterest> interests = interestMapper.selectList(
            new QueryWrapper<UserInterest>().eq("user_id", userId)
                .orderByDesc("score").last("LIMIT 10")
        );

        if (interests.isEmpty()) {
            return hotRecommend(page, size);  // 冷启动走热门
        }

        // 3. 按兴趣标签查视频
        Set<Long> videoIds = new LinkedHashSet<>();
        for (UserInterest interest : interests) {
            List<Video> videos = videoMapper.selectList(
                new QueryWrapper<Video>()
                    .like("tags", interest.getTag())
                    .eq("audit_status", 1)
                    .eq("status", 0)
                    .orderByDesc("upload_time")
                    .last("LIMIT 50")
            );
            for (Video v : videos) {
                videoIds.add(v.getId());
            }
        }

        // 4. 混入20%新视频
        List<String> freshIds = redisTemplate.opsForList().range("fresh_videos", 0, videoIds.size() / 4);
        if (freshIds != null) {
            for (String id : freshIds) {
                videoIds.add(Long.valueOf(id));
            }
        }

        // 5. 转List并缓存
        List<String> idList = videoIds.stream().map(String::valueOf).collect(Collectors.toList());
        redisTemplate.opsForList().rightPushAll(cacheKey, idList);
        redisTemplate.expire(cacheKey, 5, TimeUnit.MINUTES);

        // 6. 分页返回
        int from = (page - 1) * size;
        List<String> pageIds = idList.stream().skip(from).limit(size).collect(Collectors.toList());
        return fetchVideosByIds(pageIds);
    }

    public List<VideoVO> hotRecommend(int page, int size) {
        Set<ZSetOperations.TypedTuple<String>> tuples = redisTemplate.opsForZSet()
            .reverseRangeWithScores("hot_videos:all", (page-1)*size, page*size-1);

        List<String> ids = tuples.stream()
            .map(ZSetOperations.TypedTuple::getValue)
            .collect(Collectors.toList());
        return fetchVideosByIds(ids);
    }
}
```

## 10 参数调优表

| 参数 | 当前值 | 调整影响 |
|:---|:---:|:---|
| 点赞权重 | 3 | 越高，高点赞视频排名越前 |
| 收藏权重 | 4 | 越高，高质量内容越受推荐 |
| 评论权重 | 2 | 越高，互动性强的视频越受推荐 |
| 时间衰减指数 | 1.5 | 越大，新视频优势越明显 |
| 兴趣score上限 | 100 | 防止单一标签垄断 |
| 推荐缓存TTL | 5分钟 | 越短，推荐越实时，但DB压力越大 |
| 热度刷新周期 | 10分钟 | 越短，热度榜越实时 |
| 新视频混入比例 | 20% | 越高，新内容曝光越多 |
